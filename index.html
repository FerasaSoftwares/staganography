LSB steganography explained
01

Every pixel stores colour as numbers

Each pixel has three channels — Red, Green, Blue — each a value from 0 to 255. That's 8 bits per channel. A 500×400 image has 200,000 pixels × 3 channels = 600,000 bytes of raw data.

02

The last bit is almost invisible

Bit 7 (most significant) contributes 128 to the value. Bit 0 (least significant) contributes only 1. Changing a pixel from 200 to 201 is invisible to the human eye.

Original R=200  →  11001000  (LSB = 0)
Modified R=201  →  11001001  (LSB = 1, data hidden!)
Visual difference: completely invisible ✓
03

Message is converted to binary and written bit by bit

Each character of your message becomes 8 bits (ASCII). Those bits are written one-per-pixel-channel into the LSB. The word Hi = 01001000 01101001 = 16 bits = only 6 pixels needed.

04

Why PNG and not JPEG?

PNG is lossless — every pixel is stored exactly. JPEG uses lossy compression that changes pixel values to reduce file size, which destroys the hidden bits. Always save as PNG when using steganography.

05

How is steganography detected?

Normal images have natural variation in LSBs — roughly 50% ones and 50% zeros, but with patterns. Hidden data introduces statistical uniformity. Tools like StegDetect and zsteg look for these anomalies using chi-square analysis.

06

Real-world use

Malware has used steganography to hide command-and-control instructions inside innocent-looking images posted on social media. The malware downloads the image, reads the LSBs, and receives orders — all while traffic looks like normal image browsing.
